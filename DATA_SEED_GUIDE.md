# 银龄智护 Agent 平台数据增强与测试指南

## 一、目的

本指南用于帮助你快速增加测试数据，提升项目在以下方面的可测试性：

- 页面展示完整度
- 风险预警数量与覆盖率
- 问答升级任务的可演示性
- 指挥中心统计数据的丰富度
- 系统整体稳定性与流程闭环

---

## 二、推荐数据增强思路

为了更好地测试系统，建议至少准备以下几类数据：

### 1. 基础老人数据
用于支撑老人管理、健康档案、问答、预警等模块。

### 2. 高风险老人数据
用于测试风险预警、任务升级、运行台执行和指挥中心治理。

### 3. 实时健康异常数据
用于测试健康档案页的实时监测、建议输出和联动处置。

### 4. Agent 演示任务数据
用于测试问答升级到运行台的闭环链路。

---

## 三、现有种子脚本说明

项目当前已有以下数据脚本：

- `backend/src/scripts/seedChinaElderlyData.ts`
- `backend/src/scripts/seedMoreElderlyData.ts`
- `backend/src/scripts/seedHighRiskData.ts`
- `backend/src/scripts/seedAgentDemoBundle.ts`

### 3.1 `seedChinaElderlyData.ts`
适合导入一批较完整、较真实的基础老人数据。

### 3.2 `seedMoreElderlyData.ts`
适合做增量扩充，增加更多测试老人样本。

### 3.3 `seedHighRiskData.ts`
适合生成异常健康监测数据，用于预警测试。

### 3.4 `seedAgentDemoBundle.ts`
适合一键生成 Agent 演示包，自动注入健康数据并生成任务。

---

## 四、推荐导入顺序

为了让数据层次更合理，建议按照以下顺序导入：

### 第一步：导入基础老人数据
```bash
cd backend
npm run seed:china
```

### 第二步：扩充老人样本
如果你希望测试更多老人数据量，可以执行：

```bash
cd backend
npm run seed:more-elderly
```

### 第三步：导入高危样例数据
这一步用于增强预警、健康档案和指挥中心的展示效果。

```bash
cd backend
npm run seed:high-risk
```

### 第四步：导入 Agent 演示包
这一步会自动注入健康异常和派单任务，是最适合做答辩演示的一步。

```bash
cd backend
npm run seed:agent-bundle
```

---

## 五、推荐的测试数据规模

### 建议最小规模
- 老人数据：10~20 条
- 高风险预警：3~5 条
- 健康异常记录：10 条以上
- 可执行任务：5 条以上

### 建议演示规模
- 老人数据：20~50 条
- 预警数据：8~15 条
- 实时健康点：20 条以上
- Agent 任务：10 条以上

### 建议答辩规模
- 有基础数据、有高危数据、有任务数据、有已完成闭环数据
- 这样页面会更饱满，流程会更完整

---

## 六、导入数据后你能测试什么

### 6.1 智能问答
可以测试：
- 查询类问题
- 建议类问题
- 风险判断类问题
- 是否需要升级任务

### 6.2 风险预警
可以测试：
- 预警列表是否有数据
- 风险趋势是否有图
- 处理轨迹是否完整
- 预警是否能联动运行台

### 6.3 健康档案
可以测试：
- 实时监测建议是否生成
- 风险信号是否能识别
- 医疗知识推荐是否正常

### 6.4 运行台
可以测试：
- 是否能生成任务
- 是否能接住问答上下文
- 是否能审批执行
- 是否有事件流

### 6.5 指挥中心
可以测试：
- 闭环率是否有数值
- 任务来源是否有统计
- 风险热力是否有数据
- 模块统计是否饱满

---

## 七、推荐用于测试的演示问题

你可以使用以下问题验证系统：

- 张大爷血压高怎么办？
- 李奶奶连续两天没签到，怎么处理？
- 80 岁以上独居老人有哪些？
- 最近有哪些高风险老人？
- 这个预警需要怎么处置？

这些问题可以帮助测试：
- 问答能力
- 风险识别
- 任务升级
- 页面联动

---

## 八、如果你希望数据更多

你可以继续增加以下数据类型：

### 8.1 老人基础信息
增加更多不同地区、不同年龄、不同健康状态的老人样本。

### 8.2 健康监测点
为每位老人增加多天、多时段监测记录。

### 8.3 预警记录
制造不同风险等级的预警：
- 高血压
- 低血糖
- 发热
- 跌倒风险
- 睡眠异常

### 8.4 任务记录
增加不同状态的任务：
- 待审批
- 执行中
- 追踪中
- 已完成
- 已驳回

---

## 九、建议的数据测试目标

你现在想增加数据，主要是为了测试：

1. 页面在数据量增加后的稳定性；
2. 预警和任务是否能正常联动；
3. 运行台和指挥中心是否能显示足够丰富的数据；
4. Agent 闭环是否完整。

因此建议至少保证：
- 老人数据不低于 20 条；
- 风险数据不低于 8 条；
- 任务数据不低于 5 条；
- 有至少 1 条完整闭环演示链路。

---

## 十、总结

如果你的目标是“增加数据来测试项目稳定性和功能完成度”，最有效的方式就是：

1. 先导入基础老人数据；
2. 再导入高风险健康数据；
3. 再导入 Agent 演示包；
4. 最后在前端逐个模块验证联动效果。

这样既能提升展示完整度，也能更全面地验证系统功能。
