# 银龄智护 Agent 平台演示脚本

## 一、演示目标

展示本项目如何将养老业务中的“问题理解、任务路由、执行编排与闭环治理”整合成一个 Agent 平台。

---

## 二、演示时长建议

- 总时长：8~10 分钟
- 开场介绍：1 分钟
- 核心功能演示：6~7 分钟
- 总结升华：1 分钟

---

## 三、演示流程

### 1. 开场介绍（1分钟）

可直接这样说：

> 各位评委老师好，我们团队开发的是“银龄智护 Agent 平台”。
> 这个平台面向社区养老场景，核心不是单纯展示数据，而是把智能问答、风险预警、任务路由、执行编排和闭环治理串成一个完整工作流。

重点说明：
- 不是普通管理后台
- 不是纯聊天机器人
- 而是养老场景 Agent 平台

---

### 2. 智能问答演示（2分钟）

操作步骤：
1. 打开 `智能问答`
2. 输入问题，例如：
   - “张大爷血压高怎么办？”
   - “80岁以上独居老人有哪些？”
   - “这个风险要不要处理？”
3. 展示系统先回答，再判断是否升级任务
4. 展示最近会话、traceId、建议动作

讲解重点：
> 系统不是只返回表格，而是先给出自然语言回答，再判断是否需要进入任务处理。

---

### 3. 运行台演示（2分钟）

操作步骤：
1. 从问答页点击“去运行台创建任务”
2. 展示运行台自动带入问题上下文
3. 演示任务创建、审批、执行、事件流追踪
4. 展示任务来源与记忆信息

讲解重点：
> 问答页和运行台之间是打通的，系统会把问题上下文自动传递到执行层。

---

### 4. 风险预警演示（1.5分钟）

操作步骤：
1. 打开风险预警页
2. 展示预警统计、趋势图和预警列表
3. 打开某条预警详情
4. 演示处理预警并填写回访信息
5. 展示可直接跳转问答或运行台

讲解重点：
> 风险预警不是终点，而是 Agent 闭环的入口之一。

---

### 5. 老人管理与健康档案演示（1.5分钟）

操作步骤：
1. 打开老人管理页，展示老人列表和筛选条件
2. 演示“问 Agent / 生成任务”入口
3. 打开健康档案页，展示实时健康监测与通俗建议
4. 演示健康异常如何联动到问答或运行台

讲解重点：
> 人员管理、健康档案都能直接进入 Agent 闭环。

---

### 6. 指挥中心演示（1分钟）

操作步骤：
1. 打开指挥中心
2. 展示任务总量、SLA、闭环率、来源统计
3. 展示风险热力列表

讲解重点：
> 指挥中心负责统一看全局，不只看单条任务，而是看治理结果。

---

## 四、总结话术

可直接使用：

> 这个项目把养老业务里的问答、预警、任务、执行和治理做成了一个完整的 Agent 闭环。它既能帮助基层工作人员更快理解问题，也能把问题直接升级为任务并追踪结果，真正体现了 Agent 在民生场景中的落地价值。

---

## 五、演示注意事项

### 建议准备
- 准备好测试账号
- 准备几条预警和老人数据
- 确保后端服务和前端服务正常启动
- 提前准备好问答测试语句

### 演示技巧
- 先讲“平台定位”，再讲“功能流程”
- 尽量用一条问题串起问答、运行台、指挥中心
- 强调“闭环”这个概念
- 不要只讲功能，要讲价值

---
