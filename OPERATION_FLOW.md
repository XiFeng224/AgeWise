# 银龄智护 Agent 平台操作与数据准备文档

## 一、文档说明

本文档将 **系统操作流程** 与 **数据初始化/测试数据增强** 合并说明，帮助使用者快速完成以下目标：

- 理解平台的完整操作路径；
- 快速导入足够的数据用于稳定性测试；
- 更好地展示智能问答、风险预警、任务运行和指挥治理的闭环效果；
- 为答辩演示和 GitHub 展示准备一套可直接执行的流程。

---

## 二、平台操作总流程

建议按照以下顺序使用系统：

1. 登录系统
2. 进入首页查看平台概览
3. 使用智能问答进行问题分析
4. 如需处置，升级到 Agent 运行台
5. 在运行台创建、审批并执行任务
6. 在指挥中心查看闭环与治理情况
7. 在风险预警、老人管理、健康档案和通知中心中继续联动处理

---

## 三、详细操作步骤

### 3.1 登录系统

打开前端页面后，使用账号密码登录系统。登录成功后会进入首页 `Dashboard`。

---

### 3.2 首页概览

首页主要展示：
- 平台定位
- Agent 执行闭环
- 最近任务回执
- 快捷入口

建议先从首页了解当前系统运行状态，再进入具体业务模块。

---

### 3.3 智能问答操作

进入 `智能问答` 页面后，可以直接输入养老场景问题，例如：

- 张大爷血压高怎么办？
- 李奶奶连续两天没签到，怎么处理？
- 80 岁以上独居老人有哪些？

操作后系统会：
1. 识别问题意图；
2. 返回自然语言回答；
3. 判断是否需要升级任务；
4. 如果需要，会显示建议动作和 traceId；
5. 用户可一键跳转运行台继续处理。

---

### 3.4 风险预警操作

进入 `风险预警` 页面后，可以：
- 查看预警列表；
- 按状态、风险等级、时间范围筛选；
- 查看预警详情；
- 填写处理备注；
- 填写回访时间和回访结果；
- 直接跳转到智能问答或运行台进行处置。

---

### 3.5 老人管理操作

进入 `老人管理` 页面后，可以：
- 查看老人基础信息；
- 按姓名、风险、健康状态筛选；
- 新增、编辑、删除老人信息；
- 对特定老人直接发起问答或任务创建。

推荐操作方式：
1. 选择一个高风险老人；
2. 点击“问问 Agent”；
3. 或点击“生成任务”直接进入运行台；
4. 继续执行闭环。

---

### 3.6 健康档案操作

进入 `健康档案` 页面后，可以：
- 查看健康记录；
- 录入实时监测数据；
- 查看系统生成的通俗建议；
- 语音播报建议；
- 直接跳转到智能问答或运行台进行联动。

适合演示的监测项包括：
- 心率
- 血压
- 血糖
- 体温
- 步数
- 睡眠

---

### 3.7 通知中心操作

进入 `通知中心` 页面后，可以：
- 查看系统通知；
- 标记已读；
- 删除通知；
- 跳转指挥中心；
- 跳转运行台。

通知中心建议用于展示：
- 预警消息
- 任务消息
- 系统提醒

---

### 3.8 Agent 运行台操作

进入 `Agent 运行台` 页面后，可以：
- 选择老人；
- 填写事件摘要；
- 设置策略模式；
- 设置风险等级；
- 创建任务；
- 审批执行；
- 查看事件流；
- 导出任务演示数据。

运行台是整个系统的执行核心，适合演示完整任务流。

---

### 3.9 指挥中心操作

进入 `Agent 指挥中心` 页面后，可以：
- 查看待处理总量；
- 查看 SLA 超时；
- 查看闭环率；
- 查看服务完成率；
- 查看任务来源统计；
- 查看模块统计；
- 查看风险热力列表。

指挥中心适合向评委展示“平台治理能力”。

---

## 四、数据导入与增强

如果你希望测试项目稳定性和功能完成度，建议先导入足够的数据，再逐个页面验证联动。

### 4.1 现有数据脚本

项目中已提供以下脚本：

- `backend/src/scripts/seedChinaElderlyData.ts`
- `backend/src/scripts/seedMoreElderlyData.ts`
- `backend/src/scripts/seedHighRiskData.ts`
- `backend/src/scripts/seedAgentDemoBundle.ts`

### 4.2 推荐导入顺序

#### 第一步：导入基础老人数据
用于保证系统有足够的老人样本。

```bash
cd backend
npm run seed:china
```

#### 第二步：扩充老人样本
用于增加列表数量和筛选测试样本。

```bash
cd backend
npm run seed:more-elderly
```

#### 第三步：导入高风险样例数据
用于增强预警、健康档案和指挥中心展示。

```bash
cd backend
npm run seed:high-risk
```

#### 第四步：导入 Agent 演示包
这一步会自动注入健康异常和派单任务，是最适合答辩演示的一步。

```bash
cd backend
npm run seed:agent-bundle
```

---

## 五、推荐数据规模

### 最小可测试规模
- 老人数据：20 条左右
- 风险数据：5 条以上
- 任务数据：3 条以上

### 建议演示规模
- 老人数据：30~50 条
- 预警数据：8~15 条
- 实时健康点：20 条以上
- Agent 任务：10 条以上

## 六、导入数据后可测试内容

### 6.1 智能问答
测试自然语言理解、回答能力、任务升级能力。

### 6.2 风险预警
测试预警列表、趋势图、处置轨迹和运行台联动。

### 6.3 老人管理
测试筛选、联动 Agent、生成任务的能力。

### 6.4 健康档案
测试实时监测、通俗建议和风险判断。

### 6.5 通知中心
测试消息展示、跳转任务和治理入口。

### 6.6 运行台
测试任务创建、审批、执行和事件流。

### 6.7 指挥中心
测试闭环率、来源统计、风险热力和治理效果。

## 九、总结

银龄智护 Agent 平台的操作流程不是单一功能点的堆叠，而是一条完整的养老业务闭环。通过问答、预警、运行和治理的联动，系统能够更贴近实际社区养老工作场景。配合合理的数据导入，可以更全面地测试系统稳定性、功能完成度和演示效果。
