# 社区养老智能数据平台（人工智能应用赛道）

## 1. 项目定位

本项目面向社区养老场景，构建“**数据汇聚 + 风险识别 + 处置闭环 + 智能问答**”的一体化平台，解决社区养老中“发现慢、响应慢、追踪难、复盘弱”的问题。

项目核心目标：
- 让网格员从“被动处理”转为“主动预警”
- 让管理者从“经验决策”转为“数据决策”
- 让系统从“功能堆叠”转为“闭环治理”

---

## 2. 参赛价值与创新点

### 2.1 场景创新
- 深度贴合社区养老日常流程：老人档案、健康监测、预警、处置、回访、审计。
- 将“预警处理”从单点通知升级为全流程闭环管理。

### 2.2 AI 应用创新
- 支持自然语言查询（如“80岁以上独居老人情况”）。
- 增强**可解释AI**输出：风险等级、置信度、风险评分、关键因子与因子权重、处置建议。

### 2.3 工程创新
- 支持 Demo/Live 双模式，保证比赛现场稳定演示与真实数据切换。
- 实现预警状态流转校验与角色权限边界控制。
- 增加预警审计日志（谁在何时做了什么），可追溯、可监管。

## 4. 核心功能清单（比赛展示重点）

### 4.1 老人数据管理
- 老人档案维护（基础信息、健康信息、风险等级）
- 多维筛选检索（年龄、性别、健康状态、风险等级、地址）

### 4.2 风险预警管理（闭环）
- 预警列表与统计总览（待处理/处理中/已解决）
- 近7天趋势、风险分布可视化
- 结构化处置：状态流转、处理备注、回访时间、回访结果
- 详情与轨迹：查看完整处置轨迹（含操作人、时间、状态变化）

### 4.3 智能数据查询（AI）
- 自然语言查询
- 高级搜索
- AI 可解释结果卡：
  - 风险评分（0-100）
  - 风险等级与置信度
  - 关键因子与因子权重
  - 建议动作

### 4.4 统计分析
- 年龄分布、健康状态分布、风险分布
- 服务趋势、风险趋势预测
- 月度报告
- 数据刷新时间显示与手动刷新

## 5. 关键机制设计

### 5.1 预警状态流转规则
- `pending -> processing/resolved`
- `processing -> resolved`
- `resolved` 不可逆转

### 5.2 权限边界
- 管理员：全局查看与管理
- 网格员：仅可处理自己负责老人预警
- 家属：无预警处理权限（仅查看）

### 5.3 审计追踪
- 记录每次预警处理动作：
  - 操作人
  - 操作时间
  - 状态变化
  - 备注/回访结果

## 6. 接口响应规范（统一约定）

### 6.1 成功响应
```json
{
  "success": true,
  "message": "操作成功",
  "data": {},
  "pagination": {}
}
```

### 6.2 失败响应
```json
{
  "success": false,
  "error": "错误描述"
}
```

### 6.3 推荐错误码约定（建议）
- `400` 参数错误/非法状态流转
- `401` 未认证或令牌失效
- `403` 权限不足
- `404` 资源不存在
- `409` 资源冲突
- `500` 服务器内部错误

## 8. 5分钟演示脚本（可直接照讲）

### 第1分钟：问题背景
- 社区养老存在风险发现滞后、处置链路断点、复盘难。
- 我们做的是“智能监测 + 闭环处置 + 可解释决策”平台。

### 第2分钟：总览看板
- 展示在册老人、今日预警、待处理、高风险人数。
- 展示风险趋势与分布，说明当前社区风险态势。

### 第3分钟：预警闭环
- 进入预警列表，筛选最近7天高风险。
- 打开某条预警：展示详情与历史轨迹。
- 提交处理（处理中/已解决 + 回访结果），现场刷新数据。

### 第4分钟：AI能力
- 输入自然语言问题（例如：高风险老人预警情况）。
- 展示 AI 风险评分卡、因子权重、处置建议。
- 强调“可解释”与“可执行”。

### 第5分钟：价值总结
- 对网格员：提效与减负。
- 对管理者：全局监控与精细治理。
- 对社区：风险前置、服务提质、治理可追溯。

## 9. 项目亮点总结（答辩结尾可直接使用）

1. 不是单功能系统，而是“预警-处置-回访-审计”完整闭环。  
2. 不是黑盒AI，而是“风险评分+因子权重+建议动作”的可解释AI。  
3. 不是演示型原型，而是具备权限边界、日志追踪和工程化可落地能力。
