# 银龄智护 Agent 平台：面向社区养老的智能问答、任务路由与闭环治理系统

## 摘要

随着人口老龄化程度不断加深，社区养老服务面临着老人信息分散、风险发现滞后、服务处置链路长以及管理治理不统一等问题。针对上述痛点，本文设计并实现了一套面向社区养老场景的 Agent 平台——银龄智护 Agent 平台。系统以自然语言问答为入口，以任务编排与执行为核心，以风险预警和治理总控为闭环，构建了“问题理解—智能回答—任务升级—执行追踪—结果回写”的完整业务流。平台采用前后端分离架构，集成了千问、Python NLP Agent 以及本地规则兜底能力，并通过上下文传递与 traceId 贯通实现跨页面、跨模块的连续协同。实验与演示结果表明，该系统能够有效提升养老业务场景中的响应效率、任务处理效率与整体治理能力。

## 关键词

社区养老；Agent 平台；智能问答；任务路由；风险预警；闭环治理

## 1. 引言

### 1.1 研究背景

随着我国老龄化进程不断加快，社区养老已成为养老服务体系的重要组成部分。社区养老服务通常涉及老人基础信息管理、健康档案管理、风险预警、随访处置、通知联动和服务闭环等多种业务。传统信息系统往往侧重于数据登记和静态查询，缺少面向实际场景的智能理解与任务协同能力，导致基层工作人员在面对老人异常情况时仍需人工研判、人工派单和人工追踪，工作效率和响应速度受到限制。

### 1.2 研究目标

本研究旨在设计一套适用于社区养老场景的 Agent 平台，使系统能够：
- 理解用户的自然语言问题；
- 先提供智能回答，再决定是否升级为任务；
- 将问答结果自动路由到任务运行台；
- 支持任务规划、审批、执行和事件追踪；
- 通过指挥中心统一查看风险、来源和闭环情况。

### 1.3 研究意义

该系统将 Agent 技术从通用对话场景延伸到社区养老业务场景，具有以下意义：
- 提升社区养老服务的智能化水平；
- 降低基层工作人员的使用门槛；
- 加快风险事件响应速度；
- 打通问题发现、任务执行和结果治理的完整链路。

## 2. 系统需求分析

### 2.1 业务需求

社区养老业务主要包括以下方面：
- 老人基础档案管理；
- 健康记录和实时监测；
- 风险预警与处置；
- 通知消息分发；
- 任务流转与回执追踪；
- 管理层统一治理与统计分析。

### 2.2 功能需求

系统需要支持：
1. 自然语言提问与智能回答；
2. 风险问题自动升级任务；
3. 任务创建、审批、执行与回写；
4. 预警管理与处置轨迹查看；
5. 健康档案管理与通俗建议输出；
6. 通知中心与治理中心联动；
7. 全局上下文和 traceId 贯通。

## 3. 系统设计

### 3.1 总体架构

系统采用前后端分离架构，包含以下层次：

- **前端层**：负责页面展示、交互与任务入口承载；
- **后端层**：负责业务接口、权限控制与数据管理；
- **Agent 层**：负责意图识别、回答生成、任务路由与执行；
- **治理层**：负责任务总览、风险分布与闭环管理。

### 3.2 模块划分

#### 3.2.1 门户层 `Dashboard`
用于展示平台定位、Agent 闭环概览和入口导航。

#### 3.2.2 问答层 `DataQuery`
支持自然语言问答、意图识别、建议动作输出和任务升级。

#### 3.2.3 风险层 `RiskWarning`
展示预警信息、趋势统计、风险分布和处置轨迹。

#### 3.2.4 资源层 `ElderlyManagement` 与 `HealthRecords`
用于管理老人基础信息和健康档案，并支持任务联动。

#### 3.2.5 消息层 `Notifications`
用于承载通知、预警和系统消息，并提供跳转入口。

#### 3.2.6 执行层 `AgentVNext`
承担任务创建、执行、事件追踪与结果回写。

#### 3.2.7 治理层 `AgentCommandCenter`
用于查看任务统计、来源分布、闭环率和风险热力信息。

### 3.3 Agent 设计

本项目将 Agent 定义为具有以下四种能力的智能体：
- **理解能力**：识别用户意图；
- **回答能力**：先给出自然语言结论；
- **行动能力**：必要时升级为任务并触发执行；
- **追踪能力**：记录过程、回写结果并参与治理。

## 4. 系统实现

### 4.1 前端实现

前端采用 React + TypeScript + Ant Design 构建，页面包括：
- 平台门户首页；
- 智能问答页；
- 风险预警页；
- 老人管理页；
- 健康档案页；
- 通知中心；
- Agent 运行台；
- Agent 指挥中心。

页面之间通过统一导航、上下文存储和参数传递实现联动。

### 4.2 后端实现

后端采用 Node.js + Express + TypeScript 构建，提供：
- 身份认证与权限控制；
- 老人、预警、健康档案和通知接口；
- 智能问答接口；
- Agent 任务路由与事件流接口。

### 4.3 Agent 核心实现

系统在自然语言查询中优先调用模型路由能力，支持千问、NLP Agent 与规则兜底。问答结果会生成：
- `answer`：自然语言回答；
- `shouldEscalate`：是否需要升级任务；
- `suggestedAction`：建议动作列表；
- `traceId`：请求追踪标识。

当需要升级为任务时，运行台会接收来自问答页的上下文，并完成任务编排和执行。

## 5. 关键流程设计

### 5.1 问答到任务流程
1. 用户在问答页输入问题；
2. 系统识别意图并返回回答；
3. 系统判断是否升级任务；
4. 如需升级，自动保存上下文并跳转运行台；
5. 运行台创建任务并执行；
6. 指挥中心记录闭环结果。

### 5.2 风险处置流程
1. 风险预警页发现异常；
2. 用户查看详情或跳转问答；
3. 用户也可直接跳转运行台处置；
4. 处置结果同步至治理中心。

### 5.3 健康档案流程
1. 录入健康指标；
2. 系统生成通俗建议；
3. 若出现异常可联动风险预警或任务执行。

---

## 6. 系统创新点

### 6.1 场景化 Agent 设计
不是把 Agent 用在通用聊天，而是直接嵌入养老业务流程。

### 6.2 问答优先的任务升级机制
系统先回答，再判断是否需要执行，比传统查询更符合实际工作流。

### 6.3 任务来源可追踪
任务可标记来自问答、预警或手动创建，便于治理分析。

### 6.4 闭环治理
系统不是停留在问题识别，而是最终回到结果追踪与总控治理。

---

## 7. 结论

银龄智护 Agent 平台将自然语言问答、养老业务处理和任务闭环治理整合为统一系统，能够有效提升社区养老服务的智能化、协同化与可追踪能力。该系统不仅具备实际演示价值，也为 Agent 技术在民生场景中的落地提供了可参考的实现路径。
